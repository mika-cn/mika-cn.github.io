准备工作

1）开启第三方消息

请到 设置页面 > 高级 进行开启

2）选择 js 代码的运行方式

  • 方式一：使用 「油猴脚本」（洋文叫：userScript）浏览器扩展。
  • 方式二：自己编写浏览器扩展。

方式一很方便，你需要安装一个可以运行油猴脚本的扩展，常见的有 Greasemonkey、 Tampermonkey， 选一个适用你浏览器的安装即可。然后编写「油猴脚本」，该方式需要你对「油猴脚本」有所了解。

当然，你也可选择方式二，编写浏览器扩展可以给你最大的可控性。

与 MaoXian 交互

监听 MaoXian 发出的事件

MaoXian 会在特定的时间点，发送特定的事件，我们可以监听这些事件，并运行我们的代码。如下：

document.addEventListener('mx-wc.selecting', function(e) {
  // 注：有的事件会带有消息，消息为 JSON 字符串
  // const msg = JSON.parse(e.detail);

  // Do your work
})

发送事件给 MaoXian

下方例子，预设置当前域名和日期为tag。

document.addEventListener('mx-wc.selecting', function(e) {

  // 1. 构造消息
  const t = new Date();
  const date = [t.getFullYear(), t.getMonth() + 1, t.getDate()].join('-');
  const tagstr = [window.location.host, date].join(',');
  const msg = {options: {tagStr: tagstr}};

  // 2. 创建事件（注意：我们把消息转成 JSON 字符串）
  const detail = JSON.stringify(msg);
  const evName = 'mx-wc.set-form-input';
  const ev = new CustomEvent(evName, {detail: detail});

  // 3. 发送事件
  document.dispatchEvent(ev);
});

事件详细说明

流程图

------> ready
          |
      +-----> actived* -->  selecting --> selected --> confirmed --> clipping --> clipped* --> saving --> completed

    idle <----------------------+-------------+------------+---------------------------------------------------------+

概览

事件名 含消息？ 简介 备注
mx-wc.ready 否 初始化完成 注意：该事件会发出多次，详情见下文
mx-wc.idle 否 切回空闲状态 由其他活跃状态，进行退出或取消操作后，发出该事件
mx-wc.selecting 否 进入自由选择模式 点击「裁剪」或者从选中模式按 ESC，会触发该事件
mx-wc.selected 否 选中某个选区 点击选中某个区域后，发出该事件
mx-wc.confirmed 否 确认某个选区 在选中区域后，按 Enter 建，发出该事件
mx-wc.submitted 是 表单提交 表单提交时，触发该事件
mx-wc.clipping 否 开始裁剪 填写完表单后，点击保存后，发出该事件
mx-wc.clipped 是 裁剪完成 裁剪完成时，发出该事件
mx-wc.saving 否 存储中 存储操作开始时，发出该事件
mx-wc.completed 是 裁剪和存储完成 裁剪和存储完成后，发出该事件

mx-wc.ready

为了避免因为代码运行顺序不同，而导致事件丢失，该事件会发出多次，我们在监听该事件时，需要避免多次执行。如下：

const evName = 'mx-wc.ready';
const listener = function(e) {
  // Do your work

  // 确保该监听器，只执行一次
  document.removeEventListener(evName, listener);
}
document.addEventListener(evName, listener);

当我们收到该事件，则表明 MaoXian 已经初始化完毕，可以与其正常通讯了。

mx-wc.idle

注意：在 mx-wc.ready 过后，并不会触发 mx-wc.idle 事件。该事件只在其他活跃状态下，进行退出或取消后，才会触发，如下：

  • 用户进入自由选择模式，后进行退出
  • 用户进入选中模式后，进行退出
  • 用户确认选区后，点击表单的取消按钮
  • 裁剪完成后，触发

mx-wc.selecting

当进入「自由选择」模式时，就会触发该事件，如：

  • 当用户点击「裁剪」
  • 通过快捷键 C 进入裁剪时
  • 从「选中模式」按 ESC 退回来
  • 从填写表单，按 ESC，也会退回来

mx-wc.selected

当进入「选中模式」时，触发该事件，也就是你点击某个选区后，就会触发。

mx-wc.confirmed

从「选中模式」按下 Enter 后，会触发该事件。

mx-wc.clipping

点击表单的保存按钮后，开始裁剪的时候会触发该事件。

mx-wc.completed

整个裁剪流程完成时，触发该事件。
