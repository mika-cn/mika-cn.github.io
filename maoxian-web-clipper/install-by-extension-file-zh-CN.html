验证校验和

命令行用户可通过下方命令，计算出校验和，再和 checksum.txt 里面的内容进行比对，若一致则表明下载的文件未经修改且是完整的。

# 通过 md5 算法计算校验和
$ md5sum maoxian-web-clipper-x.x.x.zip

# 通过 sha256 算法计算校验和
$ sha256sum maoxian-web-clipper-x.x.x.zip

# 通过 sha512 算法计算校验和
$ sha512sum maoxian-web-clipper-x.x.x.zip

安装

在 Chromium 类浏览器上安装

安装说明

当你的安装环境是 Windows + Chrome 或者 MacOS + Chrome 时，在安装扩展（crx文件）的过程中，如果这个扩展不是从「谷歌商店」下载的，就会遭到禁用且无法手动启用。Chrome 此举，以“为了你的安全“为由，直接剥夺了你安装其他扩展（不是通过「谷歌商店」上线的扩展）的能力。而这个理由显然是不成立的，通过其他渠道分发的扩展就一定是恶意扩展吗？不一定吧！而通过「谷歌商店」分发的扩展就一定是可信赖的扩展吗？也不一定吧！ 说到底，对于一个软件来说，其本身对用户是否是恶意的，根本就和分发渠道无关。从「谷歌商店」上线的扩展会经过他们的审查，即使如此，最多也只能说明从「谷歌商店」的角度（而非用户的角度）来说，这些经过审查的扩展相对较安全，并不足以作为一刀切行为的理由。基于以上，我们强烈建议你弃用 Chrome，因为它既不尊重用户，也不尊重开发者。

我们推荐你选择 Chromium 浏览器或者更进一步，选择去 Google 化的 Ungoogled-Chromium 浏览器。或者其他以 Chromium 为基础开发的尊重用户权益与隐私的浏览器。如果你现在使用的是某国内大公司开发的浏览器，我们也建议你赶紧替换掉。 虽然国内大公司开发的浏览器也大多以 Chromium 为基础，但是它们会以“根据相关法律法规“ 为由（还美其名曰：“为了你的安全”），阻止你访问某些网站，996.ICU 事件就是很好的一个例子。换句话说：它们阉割了你接收信息的渠道。哪怕这些浏览器的功能做出花来，也不要选用它们。

如果你看了上面两段话后，还是决定使用 Chrome ，那么请使用下方的『zip 包解压安装法』进行安装。

本节介绍的安装方式以 Chromium 浏览器为例，原则上适用于除了 Chrome 之外的任何以 Chromium 为基础开发的浏览器。

crx 文件拖放安装法

这是目前最推荐的安装方式，安装过后是可以自动更新的（目前已经在 Linux 和 Windows 上用 Chromium 验证过，如果你发现无法自动更新请反馈）

（0）假设你已经把 crx 文件下载下来。（如果你下载的是以 .crx.zip 结尾的文件，需先重命名，把.zip 先去除）

（1）打开 Chromium 的扩展管理页面（在地址栏直接输入： about:extensions 并回车）。

（2）打开「开发者模式」的开关。

（3）用你系统的文件浏览器，打开 crx 文件所在的文件夹。

（4）用鼠标将 crx 文件拖到 Chromium 的扩展管理页面，放开后，Chromium 会询问是否安装，点击确认，安装成功。

注： 如果第 4 步的拖放，没有触发安装，则保持「开发者模式」是开启的情况下，重启浏览器，再重新试试。

zip 包解压安装法

这是黑客喜欢的安装方式，他们不怕麻烦，喜欢研究代码，讨厌自动更新带来的不确定性。

（0）假设你已经把 zip 包下载下来。（请确认你的安装包是以 .zip 结尾，而不是以 .crx 或 .crx.zip 结尾）

（1）解压 zip 包

注意： 由于打包的时候，是直接打包的程序文件，没有打包根目录，所以先建好一个目录，再把 zip 包扔里面，再进行解压。Windows 用户如果解压不了，请换一个解压软件（如：7zip）。

命令行用户请直接用 unzip 解压：

$ unzip maoxian-web-clipper-chromium-x.x.x.zip -d maoxian-web-clipper

（2）打开 Chromium 的扩展管理页面（在地址栏直接输入： about:extensions 并回车）

（3）打开「开发者模式」的开关。

（4）点击「加载已解压文件夹」按钮，选中你第一步解压出来的文件夹

注： 此种安装方式，安装的扩展，是不会自动更新的。

在 Firefox 上安装 （使用 zip 包）

注意：当前在 Firefox 上最方便的安装方式是通过火狐的扩展中心进行安装，我们也推荐你使用该种安装方式。传送门 ，

目前 zip 包无法在普通版本上进行安装，我们推荐你使用 ESR 版本的 Firefox。ESR 版本的 Firefox 更稳定，bug 也更少，并且支持 zip 包直接安装。

（1）开启 zip 包安装的开关。在地址栏输入 about:config 并回车， 点击继续， 搜索 xpinstall.signatures.required ，并把它的值改为 false。

（2）安装。在地址栏输入 about:addons 并回车，点击左边的「扩展」选项卡，再点击右上角的小齿轮，点击从文件安装，再选择你下载下来的 zip 文件。

在 Chromium 上的更新（仅适用于 zip 包安装的用户）

（1）把 zip 包解压

（2）用解压出来的文件夹，替换原来的文件夹。你可以先把旧版本的文件夹重命名，再把新版本的文件夹移动到旧版本那里。

（3）重新加载扩展，可以通过重启浏览器，或者到扩展管理页面（about:extensions），点击对应的刷新图标。

注意： 请不要在扩展管理页面（about:extensions）上， 使用先卸载再安装的方式进行更新，因为那样会丢失扩展的配置信息。

在 Firefox 上的更新 （仅适用于 zip 包安装的用户）

Firefox 请参照本页面的安装教程，直接安装新的 zip 包，就相当于更新了。
