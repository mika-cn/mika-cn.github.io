请先避免重复

在你反馈之前，请先查找以下页面，查看是否已有对应信息。

  • 常见问题页面，传送门
  • 在项目 Issue 页面，对关键字进行搜索，避免重复的反馈，传送门

反馈 Bug 注意事项

在描述你遇到的问题时候，请尽可能详细地提供信息。我们期望你能提供以下信息，以便开发者追踪问题。

  • 当前网页的网址（如果你遇到的是裁剪过程中的 bug）。
  • 你的环境信息： 系统信息，浏览器信息，MaoXian 版本，「本地程序」版本（如果你有安装的话）
  • 你的配置信息（可以在 设置 > 重置与备份 页面导出）
  • 以及详细地描述你遇到的问题（详尽地报告你遇到的问题，非常有利于开发者重现和修复该问题）。
    • 比如：
      • 这是一定会发生的?
      • 还是偶然发生的？
      • 是在某种特定情况下发生？
      • 是只在这个网页发生呢？
      • 还是在所有网页上都发生？
      • 等等…

通过 Github 项目 issue 页面，进行反馈

项目 issue 页面 。

你需要有一个 Github 帐号。另：由于墙的恶意阻止，你的浏览器很可能没法打开这个页面，如果你无法解决，请使用其他方式。

由于万恶的墙和其他原因，作者会处理完一些问题后，上线对这些问题进行批量回复。所以回复起来会很慢。

当你在 Github issue 上反馈后，你的反馈会在当天或未来几天被作者看到，并很可能已经在处理跟进。 作者期望你能把在 Github issue 上的交流看成是电子邮件般的交流，不要期望它像聊天软件。所以请仔细地描述你的问题，反馈完后，请去做你自己的事情。 作者对所有用心的反馈都会认真对待并回复的，请给她时间。
